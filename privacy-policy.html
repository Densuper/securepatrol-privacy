Privacy Policy for SecurePatrol

Last updated: 2026-03-05

SecurePatrol is designed for security patrol operations and stores operational data locally on the device by default.

What data the app handles

  • User records (name, role, PIN hash metadata)
  • Patrol logs (timestamps, checkpoint progress, completion status)
  • Registered QR checkpoint metadata
  • Device sync metadata for peer-to-peer synchronization
  • Local audit/crash/activity logs
  • Backup metadata and encrypted backup payloads

How data is stored

  • Data is stored on-device.
  • Sensitive settings are stored using encrypted preferences where supported.
  • Backups are encrypted with admin credentials/recovery flow before export.

Data sharing

  • No cloud backend is used by default.
  • Data may be shared only between authorized app devices using local peer-to-peer sync features initiated by users/admins.

Permissions used

  • Camera: QR code scanning.
  • Internet and network state: device-to-device sync workflows.

Data retention and deletion

  • Admin users can delete app data records from management screens.
  • Uninstalling the app removes local internal storage data.
  • External backups remain where the admin chose to store them.

Security controls

  • PIN-based access controls with role restrictions.
  • Session re-auth timeout.
  • Audit logs for critical actions.

Children

SecurePatrol is not designed for children.
