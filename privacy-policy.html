Privacy Policy for SecurePatrol

Last updated: 2026-03-05

SecurePatrol is designed for security patrol operations. By default, app data is stored locally on the device.

Data we handle

  • User profile data used inside the app (for example, account display names and role)
  • Authentication data (PIN hash/salt metadata)
  • Patrol activity data (start/end times, scan timestamps, checkpoint completion state)
  • Registered checkpoint metadata (QR values and admin labels)
  • Sync pairing data (device pair codes and authorized peer IDs)
  • Local audit/security event logs
  • Encrypted backup files and backup metadata

How data is used

  • Provide patrol operation features
  • Enforce role-based access controls
  • Maintain auditability and security event history
  • Support backup/restore and authorized local sync between devices

Data sharing

  • No cloud backend is used by default.
  • SecurePatrol does not run an app server to collect or store your patrol data.
  • SecurePatrol does not upload your patrol data to developer-controlled servers.
  • No third-party advertising data sharing is used.
  • Data transfer can occur only between authorized app devices when local sync is initiated by a user/admin.

Where your data lives

  • Your operational data is stored on your device.
  • If you enable backup, encrypted backup files are stored in device-selected storage only.
  • We do not keep a copy of your app data on remote servers.

Permissions

  • Camera: QR scanning and camera-based guard workflows.
  • Internet and network state: local network peer discovery and device-to-device sync.

Storage and retention

  • Primary data is stored on-device.
  • Optional external backups are written to a folder selected by the admin.
  • Data remains until deleted by authorized users or app uninstall.
  • External backups remain in selected storage until removed.

How to request account deletion and data deletion

SecurePatrol does not require a remote account service. Account and data management is handled locally in the app.

  • Open SecurePatrol and sign in as Admin.
  • For account deletion: go to user management and remove the selected user account (except protected default admin constraints).
  • For patrol/checkpoint/log deletion: use the corresponding management screens in the app.
  • For full data removal on a device: uninstall the app and delete any exported backup files from storage.

Deletion scope: local app records are removed from the device where deletion is performed. If encrypted backup files were exported, they remain until manually removed from the backup location.

Security controls

  • PIN-based authentication and role restrictions
  • Auth rate-limiting and re-auth timeout controls
  • Encrypted preference storage where supported
  • Encrypted backup payloads (AES-GCM with key derivation)

Children

SecurePatrol is not directed to children.

Changes to this policy

This policy may be updated when app functionality changes. Material changes should be reflected before rollout.
